## Admin 서비스 예상 대기시간 자동 적용 계획

1. `src/app/admin/services/page.tsx`와 관련 스토리지 유틸을 파악해 기존 진료항목 CRUD 흐름을 이해한다.
2. CSV 파싱 레이어를 헤더 동적 추출 + 통계 전략 인터페이스 구조로 재설계한다.
3. 기본 통계 전략(평균) 구현 후 UI에 공급할 공통 데이터 모델을 정의한다.
4. 관리자 서비스 화면의 업로드/프리뷰/confirm 로직을 새 데이터 모델과 전략 선택 구조에 맞게 수정한다.
5. "자동 적용" 시 기존 항목 갱신/신규 추가 로직을 유지하되 동적 헤더 라벨을 기준으로 동작하도록 보완한다.
6. 테스트·문서·TODO를 최신 구조에 맞게 업데이트하고 `.env` 의존성 없이 테스트를 실행할 수 있는 방법을 정리한다.
