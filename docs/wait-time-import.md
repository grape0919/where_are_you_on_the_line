## CSV 기반 진료항목 예상 대기시간 갱신

- 위치: `src/app/admin/services/page.tsx`
- 목적: 환자별 대기시간 raw data(csv)를 업로드해 진료 유형별 통계 대기시간을 자동으로 도출하고 적용

### 입력 데이터 규격

| 요구사항 | 내용 |
| --- | --- |
| 헤더 | 첫 번째 행에 진료 항목명이 들어가며, 개수/이름은 CSV마다 달라질 수 있다. |
| 데이터 행 | 각 행은 한 명의 환자 기록이며, 헤더와 동일한 순서의 소요시간 값을 가진다. |
| 값 형식 | 숫자 이외 문자가 포함되어도(`15분`, `20 min`) 숫자만 추출해 사용한다. |

- 빈 헤더는 `컬럼1` 같이 자동으로 보정되며, 열 개수만 맞으면 된다.
- 숫자 이외 문자가 포함되어도(`15분` 등) 숫자만 추출해 평균을 계산한다.

### 화면 흐름

1. CSV 업로드: 파일 선택 시 `parseWaitTimeCsv`가 헤더/데이터를 검증하고 환자 수 및 열별 표본 데이터를 생성한다.
2. 프리뷰: 계산된 항목 수, 평균 대기시간, 표본 수를 카드 목록으로 표시한다.
3. Confirm 단계: `결과값 자동 적용` 버튼을 누르면 업데이트/신규 생성 개수를 안내하는 확인 섹션이 열린다.
4. 적용: 확인 시 기존 항목은 `waitTime`만 갱신되고, 존재하지 않는 항목은 `isActive=true` 상태로 신규 생성된다.
5. 결과: 반영 후 성공 메시지와 함께 서비스 목록이 즉시 새로 고침된다.

### 로직 개요

- CSV 파싱(`parseWaitTimeCsv`)과 통계 계산(`calculateWaitTimeStats`)을 분리했다.
- 통계 계산은 `WaitTimeStrategy` 인터페이스를 따르며, 기본 전략은 산술 평균(`getDefaultWaitTimeStrategy`)이다.
- 파싱 결과는 열별 표본(`samples`)을 담은 `WaitTimeDataset` 배열이며, 전략은 이를 받아 `WaitTimeStat`으로 변환한다.
- 서비스 반영 시 `label` 혹은 `value`가 동일하면 기존 항목으로 간주한다.
- 반영 내역(업데이트/신규 개수)은 UI에서 `Alert` 형태로 피드백한다.
